# La Comptabilité Analytique d'Entreprise : Composantes Complètes

La comptabilité analytique (ou **comptabilité de gestion**) est un système d'information interne qui permet d'analyser les coûts, les performances et la rentabilité de l'entreprise par activité, produit, centre ou projet. Voici tous ses éléments constitutifs.

---

## 1. 🎯 Les Objets de Coûts

C'est **ce que l'on veut mesurer**. Tout élément pour lequel on souhaite calculer un coût :

| Objet de coût             | Exemples                         |
| ------------------------- | -------------------------------- |
| **Produit / Service**     | Produit A, Prestation consulting |
| **Projet**                | Projet client X, chantier N°     |
| **Client**                | Rentabilité par client           |
| **Canal de distribution** | E-commerce, réseau physique      |
| **Zone géographique**     | Belgique, UEMOA, France          |
| **Département / BU**      | RH, Production, Commercial       |

---

## 2. 🏗️ Les Centres d'Analyse (ou Centres de Coûts)

Ils permettent de **ventiler et regrouper les charges** avant de les affecter aux objets de coûts :

### a) Centres Principaux

- Directement liés à la production ou la prestation
- Exemples : atelier de production, équipe développement, salle d'opération

### b) Centres Auxiliaires

- Servent les centres principaux (RH, Maintenance, IT, Comptabilité)
- Leurs coûts sont **redistribués** vers les centres principaux via des clés de répartition

### c) Centres de Structure

- Direction générale, direction financière
- Coûts souvent traités comme charges communes

---

## 3. 💰 La Classification des Charges

### a) Par Nature (lien avec la comptabilité générale)

- Charges d'exploitation (60 à 65 en PCMN/PCG, classe 6 SYSCOHADA)
- Charges financières
- Charges exceptionnelles

### b) Par Destination (ventilation analytique)

- Charges de **production**
- Charges **commerciales**
- Charges **administratives**
- Charges de **R&D**

### c) Par Comportement (clé de la méthode des coûts)

| Type                       | Définition                            | Exemple                         |
| -------------------------- | ------------------------------------- | ------------------------------- |
| **Charges variables**      | Évoluent avec le volume d'activité    | Matières premières, commissions |
| **Charges fixes**          | Constantes quelle que soit l'activité | Loyer, amortissements           |
| **Charges semi-variables** | Mix fixe + variable                   | Électricité, téléphone          |

### d) Charges Directes vs Indirectes

- **Directes** → affectées sans calcul à l'objet de coût (matières, main-d'œuvre directe)
- **Indirectes** → nécessitent une **clé de répartition** ou un **inducteur** (loyer, direction générale)

---

## 4. 📐 Les Méthodes de Calcul des Coûts

### a) Méthode des Coûts Complets (Full Costing)

- Intègre **toutes les charges** (fixes + variables, directes + indirectes)
- Donne le **coût de revient complet**
- Utilisée pour la **fixation des prix** et les **bilans**

### b) Méthode des Coûts Variables (Direct Costing)

- N'intègre que les **charges variables**
- Calcul de la **marge sur coût variable (MCV)**
- Formule : `MCV = CA - Charges Variables`

### c) Méthode des Coûts Marginaux

- Coût de la **dernière unité produite**
- Utilisé pour les décisions de capacité supplémentaire

### d) Méthode ABC (Activity-Based Costing)

- Coûts rattachés à des **activités**, elles-mêmes rattachées aux objets
- Inducteurs d'activité (cost drivers) : nombre de commandes, heures machine, etc.
- La plus précise pour les entreprises complexes

### e) Méthode des Coûts Préétablis (Standard Costing)

- Coûts calculés **à l'avance** (standards)
- Comparaison avec les coûts réels → **calcul des écarts**

---

## 5. 📊 Les Concepts Fondamentaux de Résultats

### a) Marge sur Coût Variable (MCV)

```
MCV = Chiffre d'Affaires − Charges Variables
```

### b) Taux de Marge

```
Taux de Marge = MCV / CA × 100
```

### c) Seuil de Rentabilité (Point Mort)

```
SR = Charges Fixes / Taux de Marge sur Coût Variable
```

→ Le CA minimum pour ne pas perdre d'argent

### d) Marge de Sécurité

```
MS = CA réel − Seuil de Rentabilité
```

### e) Levier Opérationnel

```
LO = MCV / Résultat d'exploitation
```

→ Sensibilité du résultat à une variation du CA

### f) Coût de Revient

```
Coût de Revient = Coût de Production + Coût Commercial + Coût Administratif
```

### g) Résultat Analytique

```
Résultat = CA − Coût de Revient Complet
```

## 6. 🔄 Les Étapes du Processus Analytique

```
Charges générales (comptabilité générale)
          ↓
  Tri : Incorporables / Non incorporables
          ↓
  Classification : Directes / Indirectes
          ↓
  Affectation directe + Répartition indirecte
          ↓
  Calcul des coûts par centre d'analyse
          ↓
  Imputation aux objets de coûts
          ↓
  Calcul du coût de revient
          ↓
  Calcul du résultat analytique
          ↓
  Reporting & Tableaux de bord
```

---

## 7. 📋 Les Charges Non Incorporables et Supplétives

### Charges Non Incorporables

Charges comptables **exclues** de l'analytique car non représentatives :

- Charges exceptionnelles (sinistres, amendes)
- Charges hors exploitation

### Charges Supplétives

Charges **ajoutées** à l'analytique mais absentes en comptabilité générale :

- Rémunération fictive du dirigeant propriétaire
- Intérêts des capitaux propres

---

## 8. 📈 Les Tableaux de Bord & Reporting Analytique

Éléments produits par la comptabilité analytique :

- **Compte de résultat différentiel** (par produit, par segment)
- **Tableau de répartition des charges indirectes**
- **Rapport de coûts par centre de responsabilité**
- **Analyse des écarts** (réel vs. budgété)
- **Tableau de bord de rentabilité** (par client, produit, zone)
- **Rapport Make-or-Buy** (fabriquer ou sous-traiter)

## 10. 💡 Suggestion d'Application pour CashPilot

Puisque tu développes CashPilot avec support PCMN, SYSCOHADA et PCG, voici comment intégrer ces éléments nativement :

### Architecture Suggérée

```sql
-- Axes analytiques configurables
analytical_axes: { axis_id, name, type }
-- ex: "Projet", "Client", "Département", "Zone géographique"

-- Plan analytique par entreprise
cost_centers: { center_id, company_id, axis_id, name, type }
-- type: PRINCIPAL | AUXILIAIRE | STRUCTURE

-- Lignes d'imputation analytique
analytical_lines: {
  journal_entry_id,
  cost_center_id,
  amount,
  allocation_key,   -- clé de répartition %
  is_direct         -- true/false
}

-- Calcul du seuil de rentabilité par objet
breakeven_view: { object_id, fixed_costs, variable_rate, revenue, sr }
```

### Features à Prioriser

1. **Multi-axes analytiques** (Projet + Client + Zone simultanément)
2. **Répartition automatique** des charges indirectes via règles configurables
3. **Dashboard de rentabilité** par objet de coût en temps réel
4. **Calcul automatique du seuil de rentabilité** par produit/service
5. **Comparatif Budgété vs Réel** avec analyse des écarts automatisée via n8n
