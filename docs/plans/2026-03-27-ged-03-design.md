# GED-03 Implementation Plan

> **For Codex:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a company-scoped GED workflow per document that supports review requests, approval, rejection, and signing while preserving versioning and retention behavior.

**Architecture:** Store workflow state in a dedicated Supabase table keyed by `company_id + source_table + source_id`, and load it in `useGedHub` alongside documents, versions, and retention rules. Expose the current workflow status in the GED HUB table and add a dialog for workflow actions that updates the DB through the hook. Keep workflow state document-level so it survives re-uploads and version churn.

**Tech Stack:** React 18, Vite, Supabase, Vitest, Playwright, i18next.
