## Problème
Les 3 plans comptables (BE, FR, OHADA) sont actuellement stockés uniquement dans des fichiers JSON frontend. Pour l'onboarding, ils doivent être dans la table `accounting_plans` + `accounting_plan_accounts` en tant que plans système globaux.

## Solution
1. Lire les 3 fichiers JSON pour extraire la structure des comptes
2. Créer 3 entrées dans `accounting_plans` avec `is_global = true` et `source = 'system'`
3. Insérer tous les comptes dans `accounting_plan_accounts`
4. Mettre à jour `accounts_count` pour chaque plan

Les plans :
- **Belgique** : PCMN (Plan Comptable Minimum Normalisé) - ~993 comptes
- **France** : PCG (Plan Comptable Général) - ~500 comptes
- **OHADA** : SYSCOHADA révisé - ~150 comptes
