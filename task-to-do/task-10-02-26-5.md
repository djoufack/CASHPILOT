# Task 5 - [P0] : Peupler les 3 plans système depuis les JSON existants

## Fichier(s) à modifier
- Migration SQL Supabase (project_id: `rfzvrezrcigzmldgvntz`)
- Lecture des fichiers JSON : `src/data/pcg-belge.json`, `src/data/pcg-france.json`, `src/data/pcg-ohada.json`

## Problème
Les 3 plans comptables (BE, FR, OHADA) sont actuellement stockés uniquement dans des fichiers JSON frontend. Pour l'onboarding, ils doivent être dans la table `accounting_plans` + `accounting_plan_accounts` en tant que plans système globaux.

## Solution
1. Lire les 3 fichiers JSON pour extraire la structure des comptes
2. Créer 3 entrées dans `accounting_plans` avec `is_global = true` et `source = 'system'`
3. Insérer tous les comptes dans `accounting_plan_accounts`
4. Mettre à jour `accounts_count` pour chaque plan

Les plans :
- **Belgique** : PCMN (Plan Comptable Minimum Normalisé) - ~993 comptes
- **France** : PCG (Plan Comptable Général) - ~500 comptes
- **OHADA** : SYSCOHADA révisé - ~150 comptes

## Résultat
- 3 plans système insérés dans `accounting_plans` avec `is_global = true`, `source = 'system'`
- Comptes insérés dans `accounting_plan_accounts` :
  - **PCG Français** : 271 comptes (country_code: FR)
  - **PCMN Belge** : 993 comptes (country_code: BE)
  - **SYSCOHADA Révisé** : 493 comptes (country_code: OHADA)
- `accounts_count` mis à jour pour chaque plan
- Appliqué via Supabase MCP sur le projet `rfzvrezrcigzmldgvntz`
