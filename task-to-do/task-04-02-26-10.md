# Task 10 - MOYENNE : Ameliorer la gestion des erreurs RLS

## Fichiers a modifier
- `src/hooks/useProjects.js` (lignes 33-36)
- `src/hooks/useClients.js` (lignes 34-39)

## Probleme
Les erreurs RLS (42P17, 42501) sont interceptees silencieusement et le state est mis a vide. L'utilisateur ne sait pas s'il n'a pas de donnees ou s'il manque de permissions.

## Solution
Ajouter un toast d'avertissement (warning) quand une erreur RLS est detectee, au lieu de la masquer silencieusement.

### useProjects.js
```js
if (err.code === '42P17' || err.code === '42501') {
  console.warn('RLS policy error fetching projects:', err.message);
  setProjects([]);
  toast({
    title: "Access restricted",
    description: "Some data may not be visible due to permission settings.",
    variant: "default"
  });
  return;
}
```

### useClients.js
```js
if (err.code === '42P17' || err.code === '42501') {
  console.warn('RLS policy error fetching clients:', err.message);
  setClients([]);
  toast({
    title: "Access restricted",
    description: "Some data may not be visible due to permission settings.",
    variant: "default"
  });
  return;
}
```
