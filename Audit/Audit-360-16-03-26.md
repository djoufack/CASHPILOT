## Axe 1 — Securite [6.5/10]

### Constats critiques

- **[C-SEC-01] Secrets exposes dans .env versionne** : DATABASE_URL (avec mot de passe), SUPABASE_SERVICE_ROLE_KEY, SUPABASE_ANON_KEY en clair dans le repository git. Rotation de toutes les cles requise immediatement.

### Constats eleves

- **[E-SEC-02] Wildcard CORS** : `Access-Control-Allow-Origin: '*'` dans api-gateway/index.ts et mobile-money-webhook. Restreindre aux origines autorisees.

### Points forts

- 17 policies USING(true) toutes justifiees (reference data : subscription plans, SYSCOHADA, tax rates, bank providers)
- Rate limiting robuste : sign-in 12/15min, sign-up 6/1h, MFA 8/15min, hard lock a 5 echecs avec backoff exponentiel
- Timing-safe comparison pour auth interne (XOR constant-time)
- Hierarchie de scopes API correcte (admin > write > read)
- Zero injection SQL detectee (requetes parametrees Supabase)
- Zero secret hardcode dans le code source (sauf .env)
- RLS policies couvrent 78+ migrations avec filtrage company_id

### Score justifie

6.5/10 — Le constat critique C-SEC-01 (.env versionne) penalise lourdement un axe par ailleurs solide. Sans ce probleme, le score serait 8.5/10.

## Axe 2 — Conformite ENF [9/10]

### ENF-1 : Zero donnee hardcodee [8/10 — QUASI-CONFORME]

**Violations :**

- [ENF1-01] PayrollPage.jsx : objet STATUS hardcode (draft, calculated, validated, exported)
- [ENF1-02] AbsencesPage.jsx : objet STATUS hardcode (statuts demandes de conge)

**Conformites verifiees :** 125+ hooks utilisent Supabase (.from(), .rpc()), 89 pages sans donnees hardcodees, i18n pour les labels UI.

### ENF-2 : Chaine d'ownership [10/10 — CONFORME]

- Toutes les tables metier ont `company_id UUID NOT NULL REFERENCES company(id) ON DELETE CASCADE`
- Table company a `user_id NOT NULL REFERENCES auth.users(id)`
- RLS via `resolve_preferred_company_id(auth.uid())`
- useCompanyScope hook injecte company_id sur les ecritures
- Zero violation detectee

### ENF-3 : Journalisation comptable auto [9/10 — CONFORME]

- 12+ types d'operations couverts par auto*journal*\* triggers
- Couverture : invoices, supplier_invoices, payments, expenses, payroll, training, salary changes, stock movements, bank transactions, credit notes, mobile money, intercompany
- Idempotence via IF NOT EXISTS + accounting_audit_log
- Pattern : SECURITY DEFINER, SET search_path = 'public', UPSERT
- Legere deduction pour edge cases non couverts (remboursements partiels)

### Score justifie

9/10 — Conformite exemplaire. Seulement 2 violations mineures ENF-1 rapidement reparables.
