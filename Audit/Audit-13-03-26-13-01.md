Audit CashPilot (frontend, backend, database) — constats prioritaires

[Élevé] Incohérence de schéma company vs companies dans des endpoints backend
Impact: certaines routes d’export peuvent casser en prod selon le schéma réellement déployé.
Preuves: api-v1 from('companies') (SAF-T), api-v1 from('companies') (Factur-X), MCP from('companies') (SAF-T), MCP from('companies') (Factur-X).
Référence schéma repo: usage public.company.

[Élevé] generate-recurring n’est pas atomique et ignore des erreurs DB
Impact: factures orphelines, mises à jour partielles, faux succès API.
Preuves: création facture unitaire ligne 48, insert bulk sans contrôle d’erreur ligne 107, updates batch sans contrôle d’erreur ligne 112.

[Élevé] Colonnes client probablement invalides (name) dans fonctions de relance IA
Impact: relances/suggestions vides ou en erreur silencieuse.
Preuves: ai-reminder-suggest query clients(name,email), usage client.name, payment-reminders query, fallback query.

[Moyen] Fallback relances: statut “sent” sans vérifier la réponse Resend
Impact: journaux de relance inexacts (faux positifs).
Preuves: envoi non vérifié fetch, puis log forcé status: 'sent' insert.

[Moyen] Qualité frontend: lint peu fiable (bruit massif + erreurs bloquantes)
Impact: le pipeline masque les vrais problèmes.
Preuves: règle JSX désactivée eslint config, erreurs bloquantes sur directives inutiles script 1, script 2, script 3.

[Moyen] Gouvernance DB: couverture partielle des gardes migrations
Impact: dérive possible entre sources SQL.
Preuves: guard lit uniquement supabase/migrations guard, alors que le repo contient aussi migrations legacy et migrations racine.

[Faible] OAuth callback: postMessage('\*')
Impact: surface de fuite d’info de callback vers un opener non attendu.
Preuve: wildcard origin.
